1. Introducción y Propósito

1.1 Resumen del Ejecutivo

El sistema integral de Seguridad 24/7 Ecuador ha sido diseñado para centralizar todas las operaciones críticas de la empresa en una plataforma unificada, rápida y segura. El objetivo principal es digitalizar procesos que tradicionalmente consumían tiempo excesivo, como el registro de residentes, la gestión de vehículos en conjuntos habitacionales y la coordinación del equipo técnico de campo.

1.2 Alcance del Proyecto

Este sistema abarca desde la presentación corporativa de la empresa hasta paneles administrativos de alto nivel que interactúan en tiempo real con bases de datos en la nube. Permite un control exhaustivo sobre:

  • Gestión de Conjuntos Residenciales (Activos, Direcciones, Administradores).
  • Registros Detallados de Residentes y sus respectivos vehículos asociados.
  • Monitoreo de tareas técnicas y cumplimientos de contratos.
  • Portal de autoservicio para clientes finales.

1.3 Objetivos Estratégicos

1. Eficiencia Operativa: Reducción del tiempo de búsqueda de información en un 80% gracias a los filtros dinámicos.

2. Transparencia: Acceso inmediato de los clientes a sus propios datos y servicios contratados.

3. Estandarización: Implementación de normativas de ingreso de datos (Mayúsculas forzadas) para reportes impecables.

2. Arquitectura de Software

2.1 Stack Tecnológico

La plataforma utiliza una arquitectura moderna basada en la nube, garantizando una disponibilidad del 99.9% y alta velocidad de respuesta.

Frontend (Interfaz)

HTML5, CSS3 Avanzado (Custom Properties, Flexbox, Grid) y JavaScript Vanilla (ES6+). Diseño basado en la estética de Glassmorphism.

Backend (Lógica)

Modelo Serverless utilizando los servicios de Google Firebase, lo que elimina la necesidad de mantenimiento de servidores físicos.

Base de Datos

Cloud Firestore: Base de datos NoSQL documental de baja latencia para sincronización en tiempo real entre múltiples usuarios.

Seguridad y Auth

Firebase Authentication: Gestión de identidades con tokens de sesión encriptados y permisos basados en roles (RBAC).

2.2 Flujo de Datos

[Usuario] <-> [Navegador/App] <-> [Token JWT] <-> [Firebase SDK] <-> [Firestore DB]

Cada interacción del usuario es validada por reglas de seguridad en el servidor de Firebase antes de permitir cualquier operación de lectura o escritura, asegurando la integridad de la información confidencial de los clientes.

3. Descripción de Módulos (Core)

3.1 Sitio Web Corporativo

El portal público tiene como objetivo la conversión de clientes y la exposición de la marca. Incluye secciones de:

  • Inicio: Presentación de impacto.
  • Nosotros: Historia y visión renovada de la empresa.
  • Servicios: Detalle de soluciones híbridas y virtuales.
  • Tutoriales: Material educativo descargable y visual.

3.2 Control Center (Acceso Unificado)

Un único portal de entrada (`control_center.html`) que detecta automáticamente el rol del usuario (Administrador, Técnico u Operador) y lo redirige a su panel correspondiente, simplificando la curva de aprendizaje para el personal.

3.3 Gestión Administrativa

El módulo `gestion_admin.html` permite a la gerencia:

  • Crear y editar planes de suscripción.
  • Gestionar el catálogo de clientes maestros.
  • Auditar las estadísticas generales de la empresa.

4. Gestión de Operaciones y Registros

Este es el núcleo operativo de la empresa (`gestion_operaciones.html`). Ha sido diseñado para un manejo masivo de datos con una estética premium de alto contraste (Oro/Negro).

4.1 Registro de Conjuntos

Permite dar de alta unidades habitacionales asociándolas a un administrador responsable. Cada conjunto actúa como un contenedor para los residentes.

4.2 Registro de Residentes y Vehículos

Un sistema dinámico donde se pueden agregar múltiples residentes por villa/casa y asociar uno o varios vehículos.

Campo Validación / Regla
CASA / VILLA Mayúsculas, requerido, único dentro del conjunto.
TIPO Propietario o Arrendatario (Badges visuales).
NOMBRE/APELLIDO Transformación automática a Mayúsculas.
PLACAS V. Formato estandarizado en Mayúsculas.

4.3 Funciones de Exportación

Implementación de ExcelJS para descargar en un solo clic toda la base de datos de un conjunto residencial para reportes externos o impresión física.

5. Gestión Técnica y Reportes

5.1 Control de Personal de Campo

El panel `gestion_tecnica.html` enfoca sus funcionalidades en el equipo que realiza instalaciones y mantenimientos.

Funcionalidades Clave:

  • Iniciación de Labores: Marcado de tiempo real al inicio de una jornada o trabajo.
  • Reporte de Novedades: Formulario para describir incidentes técnicos.
  • Gestión de Fotos: Subida de evidencias visuales del trabajo realizado.

5.2 Estandarización de Reportes Técnicos

Al igual que en operaciones, toda la información técnica se guarda de manera normalizada. Esto permite generar estadísticas precisas sobre qué equipos fallan con mayor frecuencia o qué conjuntos requieren más mantenimiento preventivo.

Nota Informativa: El sistema detecta automáticamente la ubicación del técnico al momento de enviar reportes para mayor transparencia operativa.

6. Portal de Clientes e Historial

La transparencia es un pilar en Seguridad 24/7 Ecuador. El `portal_clientes.html` es el espacio donde el usuario final (Administradores de conjuntos o clientes directos) puede interactuar con la empresa.

6.1 Visualización de Servicios

El cliente puede ver el estado actual de sus contratos, verificar qué técnicos han visitado sus instalaciones recientemente y descargar facturas o reportes de monitoreo.

6.2 Canales de Comunicación

Integración directa con WhatsApp y Telefonía IP para reportar emergencias en un clic. El sistema precarga la información del cliente para que la central de monitoreo sepa exactamente quién está llamando.

6.3 Seguridad del Portal

El acceso está restringido por correos electrónicos verificados, evitando que personal ajeno al conjunto residencial pueda filtrar información de los residentes.

7. Estructura de Datos y Seguridad

7.1 Modelo NoSQL Firestore

Estructura de las colecciones principales:

  • `complexes`: Almacena info de conjuntos.
  • `residents_registry`: Documentos con arrays anidados de residentes y vehículos.
  • `jobs`: Seguimiento de la actividad técnica.
  • `users`: Perfiles y niveles de permiso.

7.2 Normativa de Datos (Mayúsculas)

Para garantizar una base de datos limpia, se ha implementado un "middleware" a nivel de frontend que intercepta los datos de los formularios y ejecuta el método `.toUpperCase()` antes de enviarlos a la base de datos.
